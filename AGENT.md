# AI 协作约定

## 脚本开发

+ 当任务涉及新增、修改、删除、重命名 `script/` 下的脚本时，优先阅读 `rules/script_rule.md`
+ 编写脚本时，必须遵守 `rules/script_rule.md` 中的约束，包括命名、路径解析、`--help`、文档同步、跨平台和公共库复用规则

## 脚本执行

+ 当任务涉及执行现有脚本完成工作时，优先阅读 `script/script.md`
+ 应先根据 `script/script.md` 中的脚本用途、最小参数摘要和示例，选择最合适的脚本执行任务
+ 若 `script/script.md` 已能明确脚本用途与参数，则不要先读源码
+ 阅读顺序应为：`script/script.md` -> 脚本 `--help` -> 必要时再读脚本源码

## 通用要求

+ 优先复用现有脚本，而不是重复实现同类能力
+ 不要假设脚本必须在仓库根目录执行，执行前应确认脚本约定的调用方式
+ 对需要 `source` 或 `call` 的脚本，执行时必须使用正确方式，不能用错误解释器直接运行
