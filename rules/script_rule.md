# 脚本规约

本目录定义 `script/` 的统一约束，目标是让脚本可被人和 AI 稳定发现、理解、执行与维护。

## 阅读顺序

+ `script/script.md`
+ 对应脚本的 `--help`
+ 仅在必要时再读取脚本源码

## 规约目标

+ 脚本在不同当前工作目录下执行，默认行为与产物保持一致
+ 同一逻辑脚本在不同平台上的入口、参数与输出尽量一致
+ AI 先读索引再执行，减少直接读源码的成本
+ 最小化重复文档，详细参数以 `--help` 为准

## 强制要求

+ 新增、删除、重命名脚本时，必须同步更新 `script/script.md`
+ 变更脚本参数、默认行为、输出位置、依赖要求时，必须同步更新 `script/script.md` 与 `--help`
+ 脚本不得隐式依赖“当前正好在某个目录执行”这一前提

## 开发规则

### 实现语言

+ 优先使用 Shell 实现脚本入口与流程编排
+ 当逻辑在 Shell 中明显不稳定、不易维护或跨平台处理成本过高时，允许使用 Python 承担核心逻辑
+ 若需要同时支持 Linux 与 Windows，优先保证脚本能力一致，再选择合适入口形式
+ 同一逻辑脚本可以提供多平台入口，例如 `xxx.sh` 与 `xxx.bat`

### 跨平台要求

+ Linux 与 Windows 若都支持，参数名、参数含义、默认行为、退出语义应尽量一致
+ 平台差异无法完全消除时，应在 `--help` 中明确说明差异
+ 平台专属能力不得冒充通用能力
+ 能复用公共配置时，优先复用同一份配置文件

### 路径与执行规则

+ 脚本内部引用的仓库内文件，必须基于“脚本自身位置”或“仓库根目录”解析，不能依赖当前工作目录。如有一个读取文件的脚本：` ``readFile.sh --file config.json `，那么不论是在 `bash /home/script/readFile.sh`或者 `cd /home/script && bash readFile.sh`，输出结果需要一样。
+ 不允许写出“在根目录执行正常、切到 `script/` 执行就写到别处”的脚本

### 目录与命名

+ `script/` 存放可执行脚本与脚本直接使用的配置
+ `script/_lib/` 存放脚本公共库，不作为用户直接调用入口
+ 一个逻辑能力优先使用同名前缀命名，例如 `install_env.sh`、`install_env.bat`
+ 脚本名应直接表达动作与对象，避免使用无语义缩写

### CLI 规则

+ 每个可执行脚本都必须支持 `--help`
+ `--help` 必须能够独立说明用途、用法、参数、默认值或默认行为、必要限制
+ 详细参数说明以 `--help` 为准
+ 对需要交互输入的脚本，缺少必要参数时可以进入交互模式，但 `--help` 中必须说明
+ 若脚本支持 `--verbose`、`--dry-run`，各平台入口语义应保持一致

### 输出与退出

+ 成功与失败必须有稳定的退出码
+ 错误信息应直接说明失败原因与缺失条件
+ 输出内容优先面向自动化消费，避免无意义装饰文本
+ 当脚本会修改文件、环境变量或安装目录时，应在帮助信息中说明副作用

### 公共库复用

+ Shell 脚本优先复用 `script/_lib/common.sh`
+ Python 脚本优先复用 `script/_lib/cli.py`、`script/_lib/paths.py`、`script/_lib/logging.py`
+ 路径解析、日志初始化、通用参数等基础能力，优先沉淀到公共库，不在多个脚本中重复实现

### 变更要求

+ 新增脚本时，必须同时补充 `script/script.md`
+ 变更脚本参数或行为时，必须同步更新 `--help`
+ 若新增了可复用公共逻辑，应优先放入 `_lib/`，再由脚本引用

### 最小验收

+ 在仓库根目录执行 `script/<name> --help` 能正常返回帮助信息
+ 在 `script/` 目录执行 `./<name> --help` 能正常返回帮助信息
+ 两种执行方式下，默认配置解析结果一致
+ `script/script.md` 已同步更新

### --help 内容要求

+ Usage
+ 参数列表
+ 必填与可选项
+ 默认值或默认行为
+ 是否支持交互模式
+ 关键副作用
+ 平台差异（若有）

### 文档要求

+ `script/script.md`必须包含所有脚本的最小描述，只需要包含脚本名称，脚本作用既可。格式为：脚本分类（一级目录，没一类功能的脚本放在一个一级目录下），脚本名称[xxx.sh/xxx.bat/xxx.py]：脚本描述
